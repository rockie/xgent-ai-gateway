**4. Create gateway/src/auth/api_key.rs** with these functions and types (per D-01 through D-11):

- `ClientMetadata` struct: `{ key_hash: String, service_names: Vec<String>, created_at: String }`
- `fn generate_api_key() -> (String, String)` -- generates 32 random bytes via `rand::rng().fill_bytes()`, returns `(raw_key_hex_64chars, sha256_hash_hex)`. The raw key is a 64-char hex string. The hash is `hex::encode(Sha256::digest(raw_key.as_bytes()))`.
- `async fn store_api_key(conn: &mut redis::aio::MultiplexedConnection, key_hash: &str, service_names: &[String]) -> Result<(), redis::RedisError>` -- does `HSET api_keys:{key_hash} service_names "{comma_joined}" created_at "{utc_now_rfc3339}"`.
- `async fn lookup_api_key(conn: &mut redis::aio::MultiplexedConnection, key_hash: &str) -> Result<Option<ClientMetadata>, redis::RedisError>` -- does `HGETALL api_keys:{key_hash}`, returns None if empty, otherwise parses into ClientMetadata.
- `async fn revoke_api_key(conn: &mut redis::aio::MultiplexedConnection, key_hash: &str) -> Result<bool, redis::RedisError>` -- does `DEL api_keys:{key_hash}`, returns true if deleted.
- `fn extract_api_key(headers: &axum::http::HeaderMap) -> Option<String>` -- per D-05/D-06: checks `Authorization: Bearer <key>` first, falls back to `X-API-Key: <key>` header. Returns the raw key string.
- `fn hash_api_key(raw_key: &str) -> String` -- returns `hex::encode(Sha256::digest(raw_key.as_bytes()))`.
- Axum middleware function `pub async fn api_key_auth_middleware(State(state): State<Arc<AppState>>, mut request: axum::extract::Request, next: axum::middleware::Next) -> Result<axum::response::Response, axum::http::StatusCode>`:
  - Extracts key via `extract_api_key` from request headers
  - If missing: `tracing::debug!("API key missing from request")` then return `Err(StatusCode::UNAUTHORIZED)` (per D-10/D-11)
  - Hashes key with `hash_api_key`
  - Calls `lookup_api_key` using `state.auth_conn` (a `redis::aio::MultiplexedConnection` on AppState)
  - If not found: `tracing::debug!("API key not found in store")` then return `Err(StatusCode::UNAUTHORIZED)`
  - If found: insert `ClientMetadata` into request extensions via `request.extensions_mut().insert(meta)`, then call `next.run(request).await`
- Add `#[derive(Debug, Clone, serde::Serialize)]` on ClientMetadata for use in admin responses.

        assert_eq!(hash1, hash2, "hashing same input should yield same output");
    }

        assert_eq!(hash_api_key(&raw), hash, "hash of raw key should match generated hash");

**5. Create gateway/src/tls/config.rs** with two builder functions:

`pub fn build_http_tls_config(tls: &crate::config::TlsConfig) -> Result<rustls::ServerConfig, Box<dyn std::error::Error>>`:
- Open cert_path and key_path files
- Parse certs with `rustls_pemfile::certs(&mut BufReader::new(cert_file)).collect::<Result<Vec<_>, _>>()?`
- Parse key with `rustls_pemfile::private_key(&mut BufReader::new(key_file))?.ok_or("no private key found")?`
- Build: `ServerConfig::builder().with_no_client_auth().with_single_cert(certs, key)?`
- Set `config.alpn_protocols = vec![b"h2".to_vec(), b"http/1.1".to_vec()]` (per Pitfall 3 in research)
- Return config

`pub fn build_grpc_tls_config(tls: &crate::config::GrpcTlsConfig) -> Result<tonic::transport::ServerTlsConfig, Box<dyn std::error::Error>>`:
- Read cert, key, and client_ca files as strings with `std::fs::read_to_string`
- Build: `ServerTlsConfig::new().identity(Identity::from_pem(&cert, &key)).client_ca_root(Certificate::from_pem(&client_ca))`
- Return the ServerTlsConfig
