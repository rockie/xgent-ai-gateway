<objective>
Create Tower Service auth layers for gRPC and wire per-service authorization into all RPC handlers.

Purpose: Bring gRPC authentication to parity with HTTP -- every gRPC RPC enforces the same auth as its HTTP counterpart. This closes the critical security gap where gRPC SubmitTask, GetTaskStatus, ReportResult, Heartbeat, and DrainNode are currently unauthenticated.

Output: Two Tower auth wrapper structs in `grpc/auth.rs`, updated handlers in `submit.rs` and `poll.rs` with per-service authorization checks, and wiring in `main.rs`.
</objective>

  - Clone `self.inner` and `self.state` (Tower clone contract per Pitfall 3)
  - Extract API key from `req.headers()` using `crate::auth::api_key::extract_api_key(req.headers())` (per D-05: checks `Authorization: Bearer` first, falls back to `x-api-key`)
  - If no key found: log at debug level `"gRPC API key missing"`, increment `state.metrics.errors_total.with_label_values(&["unknown", "auth_api_key"]).inc()` (per D-14), return `tonic::Status::unauthenticated("unauthorized").into_http()` (per D-11)

  - Clone `state.auth_conn` for Redis lookup (per D-03)
  - Call `crate::auth::api_key::lookup_api_key(&mut conn, &hash).await`
  - If `Ok(Some(meta))`: insert `meta` into `req.extensions_mut().insert(meta)`, then call `inner.call(req).await`
  - If `Ok(None)`: log debug `"gRPC API key not found"`, increment `errors_total` with `["unknown", "auth_api_key"]`, return `Status::unauthenticated("unauthorized").into_http()`
  - If `Err(e)`: log error `"Redis error during gRPC API key lookup: {e}"`, return `Status::internal("internal error").into_http()`
- Constructor: `pub fn new(inner: S, state: Arc<AppState>) -> Self`

  - If invalid: log debug, increment `errors_total`, return `Status::unauthenticated("unauthorized").into_http()`
