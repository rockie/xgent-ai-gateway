**Auth enforcement (gateway/src/grpc/auth.rs):**

1. Add a helper function to compute SHA-256 fingerprint:
```rust
fn cert_fingerprint(cert_der: &[u8]) -> String {
    use sha2::{Sha256, Digest};
    let hash = Sha256::digest(cert_der);
    hex::encode(hash)
}
```

2. Add a helper function to check mTLS identity:
```rust
fn check_mtls_identity(
    config: &crate::config::MtlsIdentityConfig,
    peer_certs: &[tonic::transport::server::TlsConnectInfo<tokio_rustls::server::TlsStream<tokio::net::TcpStream>>],
    service_name: &str,
) -> Result<(), Status> {
    // ... implementation
}
```

Actually, the approach depends on how peer certs are available. In tonic, when using `ServerTlsConfig` with client auth, `request.peer_certs()` returns `Option<Arc<Vec<Certificate>>>`. Each `Certificate` has a `.get_ref()` returning `&[u8]` (DER bytes).

3. In `NodeTokenAuthLayer::call()`, after successful node token validation, if `self.state.config.grpc.mtls_identity.fingerprints` is non-empty, extract peer certs from the request and verify:

```rust
// After node token validation succeeds, check mTLS identity if configured
let mtls_config = &self.state.config.grpc.mtls_identity;
if !mtls_config.fingerprints.is_empty() {
    // Extract peer certs from tonic request extensions
    if let Some(certs) = req.extensions().get::<Arc<Vec<tonic::transport::Certificate>>>() {
        if let Some(cert) = certs.first() {
            let fp = cert_fingerprint(cert.get_ref());
            match mtls_config.fingerprints.get(&fp) {
                Some(allowed_services) => {
                    if !allowed_services.contains(&service_name.to_string()) {
                        return Box::pin(async {
                            Ok(Status::permission_denied("certificate not authorized for this service")
                                .into_http())
                        });
                    }
                }
                None => {
                    return Box::pin(async {
                        Ok(Status::permission_denied("unknown certificate fingerprint")
                            .into_http())
                    });
                }
            }
        } else {
            return Box::pin(async {
                Ok(Status::permission_denied("no client certificate presented").into_http())
            });
        }
    }
    // If no peer_certs extension, mTLS might not be active -- skip check
    // This allows plain-text dev mode to work without cert identity
}
```

IMPORTANT: The exact API for accessing peer certs depends on tonic 0.14's implementation. The executor MUST:
- Check if `tonic::transport::server::TlsConnectInfo` is available in request extensions
- Check `request.peer_certs()` method availability on tonic 0.14
- If `peer_certs()` is not available or returns None, check request extensions for certificate data
- If neither works, add a `tracing::warn!` and skip the check (per D-13, this is v1 and can be improved)

The `sha2` crate is already a dependency (`sha2 = "0.10"` in Cargo.toml for API key hashing). The `hex` crate is also already a dependency.
