### Integration Verified

3. **Auth middleware** — All admin endpoints protected, auth endpoints correctly unprotected ✓

5. **UI patterns** — ErrorAlert/EmptyState used on all pages; Skeleton variations appropriate ✓
6. **Dashboard → Service detail** — ServiceHealthList links navigate to Phase 9 detail page ✓
7. **Shared apiClient** — All 5 data layer files import from Phase 8's `api.ts` with `credentials: 'include'` ✓

### Minor Integration Gap

**`canCancel` omits `assigned` state:** `admin-ui/src/lib/tasks.ts:111` — `canCancel` returns `state === 'pending' || state === 'running'`. Backend `types.rs` permits `Assigned → Failed`. This is technically beyond the requirement text ("cancel a pending or running task") but is a UX gap since the backend supports it. Fix: add `|| state === 'assigned'`.
