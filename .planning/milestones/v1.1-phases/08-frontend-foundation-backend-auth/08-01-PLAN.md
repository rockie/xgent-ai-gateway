```rust
/// Admin endpoint configuration.
#[derive(Debug, Deserialize, Clone)]
pub struct AdminConfig {
    /// Admin username. If not set, admin endpoints are unauthenticated (dev mode).
    pub username: Option<String>,
    /// Argon2 PHC-format password hash for the admin user.
    pub password_hash: Option<String>,
    /// Allowed CORS origin for the admin SPA (e.g., "https://admin.example.com").
    pub cors_origin: Option<String>,
    /// Session TTL in seconds. Default 86400 (24 hours).
    #[serde(default = "default_session_ttl_secs")]
    pub session_ttl_secs: u64,
    /// Whether to set Secure flag on session cookie. Default true.
    #[serde(default = "default_true")]
    pub cookie_secure: bool,
}

impl Default for AdminConfig {
    fn default() -> Self {
        Self {
            username: None,
            password_hash: None,
            cors_origin: None,
            session_ttl_secs: default_session_ttl_secs(),
            cookie_secure: true,
        }
    }
}

fn default_session_ttl_secs() -> u64 {
    86400
}
```

**3. Create gateway/src/http/auth.rs** with:

- `LoginRequest` struct: `{ username: String, password: String }`
- `LoginResponse` struct: `{ username: String }`
- `generate_session_id()` function: generates 32 random bytes via `rand::Rng::random::<[u8; 32]>()`, returns hex-encoded string (64 chars)
- `verify_password(password: &str, stored_hash: &str) -> bool`: uses `argon2::Argon2::default()` and `password_hash::PasswordVerifier::verify_password` to check against PHC-format hash
- `login` handler (per D-06, API-01):
  - Accept `State<Arc<AppState>>`, `CookieJar`, `Json<LoginRequest>`
  - Compare `req.username` against `state.config.admin.username`
  - Verify password against `state.config.admin.password_hash` using `verify_password`
  - On success: generate session ID, HSET `admin_session:<id>` with fields `username` and `created_at` (RFC3339), EXPIRE with `session_ttl_secs`
  - Build cookie: name=`session`, value=session_id, path=`/`, http_only=true, secure=`state.config.admin.cookie_secure`, same_site=`SameSite::None`, max_age=`session_ttl_secs`
  - Return `(CookieJar, Json<LoginResponse>)` with 200
  - On failure: return 401 with JSON `{ "error": "Invalid username or password" }`
- `logout` handler (per D-06, AUTH-04):
  - Extract session cookie, DEL `admin_session:<id>` from Redis
  - Set cookie with max_age=0 to clear
  - Return 200
- `refresh` handler (per D-05, API-02):
  - Extract session cookie, check EXISTS in Redis
  - If exists: EXPIRE `admin_session:<id>` with `session_ttl_secs`, return 200
  - If not: return 401
- `session_auth_middleware` (replaces admin_auth_middleware):
  - If `state.config.admin.username.is_none()` -> pass through (dev mode, per D-01)
  - Extract `session` cookie from `CookieJar`
  - Check `EXISTS admin_session:<session_id>` in Redis
  - If not exists: return 401
  - If exists: EXPIRE (sliding window per D-04), call `next.run(req)`
