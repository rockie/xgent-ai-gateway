**Filter controls:**

1. **Task ID search (D-09):** `<Input placeholder="Search by task ID..." />` with debounced onChange (300ms timeout via `useRef` + `setTimeout`/`clearTimeout`). When non-empty, sets `filters.task_id`. When cleared, removes `filters.task_id`. This bypasses other filters (direct lookup mode).

2. **Service dropdown (D-07):** shadcn `<Select>` populated from `useServices()` data. Options: "All services" (value=""), plus one option per service name. OnValueChange sets `filters.service` or removes it if "All services" selected.

3. **Status multi-select (D-08):** Build using Popover + Checkbox list pattern. Trigger button shows "Status" with count badge when filters active. Popover content: list of checkboxes for Pending, Assigned, Running, Completed, Failed. Each checkbox toggles its state in a local `Set<string>`. On change, join selected states with comma and set `filters.status`. If none selected, remove `filters.status`.

4. **Page size dropdown (D-06):** shadcn `<Select>` with options 10, 25, 50. Default 25. OnValueChange sets `filters.page_size`.

**Pagination (D-05):**

Previous button: disabled when `!filters.cursor` (first page). Store cursor history in `useRef<string[]>([])` array. When Next is clicked, push current cursor to history. When Previous is clicked, pop from history and set as cursor. Next button: disabled when response `cursor` is null.

**Row click handler:**

`onRowClick={(taskId) => { setSelectedTaskId(taskId); setSheetOpen(true) }}`

**Loading/Error/Empty states:**

- Loading (first load, no data): `<PageSkeleton />` or table skeleton rows
- Error: `<ErrorAlert error={error} onRetry={refetch} />`
- Empty (no tasks at all, no filters): EmptyState with ListTodo icon, "No tasks", "No tasks have been submitted yet."
- Empty (filters active but no results): EmptyState with Search icon, "No matching tasks", "No tasks match your filters. Try adjusting your search criteria."
