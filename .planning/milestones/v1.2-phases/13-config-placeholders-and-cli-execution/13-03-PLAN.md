    1. Replace the Cli struct with a minimal version (per D-04):
       #[derive(Parser, Debug)]
       #[command(name = "xgent-agent", about = "Node-side runner agent for xgent gateway")]
       struct Cli {
           /// Path to agent.yaml config file
           #[arg(long, default_value = "agent.yaml")]
           config: String,

           /// Validate config and print resolved templates without executing
           #[arg(long)]
           dry_run: bool,
       }

    2. In main():
       (a) Parse Cli args
       (b) Call agent::config::load_config(&cli.config) — fail with clear error on load failure
       (c) If cli.dry_run: print loaded config summary (service name, mode, command, response template) and exit 0
       (d) Build executor based on config.service.mode:
           ExecutionMode::Cli => {
               let cli_section = config.cli.expect("cli section required for cli mode");
               Box::new(CliExecutor::new(config.service.name.clone(), cli_section, config.response.clone())) as Box<dyn Executor>
           }
           ExecutionMode::SyncApi | ExecutionMode::AsyncApi => {
               eprintln!("sync-api and async-api modes not yet implemented");
               std::process::exit(1);
           }
       (e) Update tracing::info! to log service name, mode, gateway address from config
       (f) Set up reconnect loop using config.gateway.max_reconnect_delay_secs

    3. Update run_poll_loop signature:
       - Remove `http_client: &reqwest::Client` parameter
       - Add `executor: &dyn Executor` parameter
       - Add `config: &AgentConfig` parameter (for gateway address, token, TLS, service name, node_id)

    4. Inside run_poll_loop:
       - Use config.gateway.address for gRPC channel (instead of cli.gateway_addr)
       - Use config.gateway.token for auth (instead of cli.token)
       - Use config.gateway.ca_cert and config.gateway.tls_skip_verify for TLS
       - Use config.service.name for PollTasksRequest.service_name
       - Use config.gateway.node_id for PollTasksRequest.node_id

    5. Replace the dispatch_task call with executor:
       BEFORE:
         let dispatch_result = dispatch_task(http_client, &cli.dispatch_url, &assignment).await;
       AFTER:
         let exec_result = executor.execute(&assignment).await;

    6. Map ExecutionResult to ReportResultRequest:
       ReportResultRequest {
           task_id: assignment.task_id.clone(),
           success: exec_result.success,
           result: exec_result.result,
           error_message: exec_result.error_message,
           node_id: config.gateway.node_id.clone(),
           service_name: config.service.name.clone(),
       }

    7. Delete the dispatch_task() function entirely (it is fully replaced)

    8. Delete `use reqwest` — no longer needed in agent binary for CLI mode

    9. Preserve ALL of these unchanged:
       - SHUTTING_DOWN AtomicBool
       - shutdown_signal() function
       - graceful_drain() function (update parameter names to use config fields)
       - The outer reconnection loop in main() with exponential backoff
       - The tokio::select! in run_poll_loop for shutdown vs message
       - in_flight_done / has_in_flight tracking

    10. Add Clone derive to ResponseSection in config.rs if not already present (needed for CliExecutor::new)
