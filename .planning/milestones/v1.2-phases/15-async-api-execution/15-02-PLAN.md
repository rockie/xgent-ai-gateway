**Add `evaluate()` method on CompletionCondition** (per D-11/D-12):
```rust
impl CompletionCondition {
    /// Evaluate the condition against a JSON response.
    /// Extracts the value at `self.path` and compares using `self.operator`.
    pub fn evaluate(&self, json: &serde_json::Value) -> Result<bool, String> {
        let actual = super::http_common::extract_json_value(json, &self.path)?;
        match (&self.operator, &self.value) {
            (ConditionOperator::Equal, ConditionValue::Single(expected)) => {
                Ok(actual == *expected)
            }
            (ConditionOperator::NotEqual, ConditionValue::Single(expected)) => {
                Ok(actual != *expected)
            }
            (ConditionOperator::In, ConditionValue::Multiple(values)) => {
                Ok(values.iter().any(|v| v == &actual))
            }
            (ConditionOperator::NotIn, ConditionValue::Multiple(values)) => {
                Ok(!values.iter().any(|v| v == &actual))
            }
            (ConditionOperator::Equal | ConditionOperator::NotEqual, ConditionValue::Multiple(_)) => {
                Err(format!("operator {:?} requires a single string value, not an array", self.operator))
            }
            (ConditionOperator::In | ConditionOperator::NotIn, ConditionValue::Single(_)) => {
                Err(format!("operator {:?} requires an array value, not a single string", self.operator))
            }
        }
    }
}
```

```rust
use std::collections::HashMap;
use std::time::Duration;

use async_trait::async_trait;
use reqwest::redirect::Policy;
use tracing::{info, warn, error};

use super::config::{AsyncApiSection, ResponseSection, CompletionCondition};
use super::executor::{ExecutionResult, Executor};
use super::http_common;
use super::placeholder;
use super::response;
use xgent_proto::TaskAssignment;

pub struct AsyncApiExecutor {
    service_name: String,
    async_api: AsyncApiSection,
    response: ResponseSection,
    client: reqwest::Client,
}

impl AsyncApiExecutor {
    pub fn new(
        service_name: String,
        async_api: AsyncApiSection,
        response: ResponseSection,
    ) -> Result<Self, String> {
        // Build reqwest::Client WITHOUT per-request timeout (per Pitfall 5 in RESEARCH.md)
        // The outer tokio::time::timeout handles total duration (D-08)
        let mut builder = reqwest::Client::builder()
            .redirect(Policy::limited(5));
        if async_api.tls_skip_verify {
            builder = builder.danger_accept_invalid_certs(true);
        }
        let client = builder.build()
            .map_err(|e| format!("failed to build HTTP client: {}", e))?;
        Ok(Self { service_name, async_api, response, client })
    }
}
```

**Executor implementation:**
```rust
#[async_trait]
impl Executor for AsyncApiExecutor {
    async fn execute(&self, assignment: &TaskAssignment) -> ExecutionResult {
        let timeout_dur = Duration::from_secs(self.async_api.timeout_secs);
        match tokio::time::timeout(timeout_dur, self.run_submit_poll(assignment)).await {
            Ok(result) => result,
            Err(_) => ExecutionResult {
                success: false,
                result: Vec::new(),
                error_message: format!("async-api timed out after {}s", self.async_api.timeout_secs),
                headers: HashMap::new(),
            },
        }
    }
}
```

**Private methods on AsyncApiExecutor:**

`send_request()` — same retry pattern as SyncApiExecutor.send_request() but uses self.client directly without timeout errors (no per-request timeout set). Retry on is_connect() only (D-13/D-14). Returns `Result<reqwest::Response, ExecutionResult>`.

`run_submit_poll()` — the core logic:
1. Build task variables from assignment via `placeholder::build_task_variables()`
2. Resolve submit URL via `placeholder::resolve_placeholders(&self.async_api.submit.url, &variables)`
3. Resolve submit body (if present) via `placeholder::resolve_placeholders()`
4. Resolve submit headers via `placeholder::resolve_placeholders()` for each header value
5. Parse submit method
6. Send submit request via `self.send_request()` with retry
7. Check submit response status — non-2xx fails immediately (D-15)
8. Parse submit response body as JSON
9. Scan poll URL template AND poll body template (if present) for `<submit_response.*>` placeholders via `http_common::find_prefixed_placeholders(template, "submit_response")`
10. Extract each submit_response path from submit JSON via `http_common::extract_json_value()` and insert into variables as `submit_response.{path}`
11. Enter poll loop:
    a. `tokio::time::sleep(Duration::from_secs(self.async_api.poll.interval_secs)).await`
    b. Resolve poll URL and body with variables (now includes submit_response values)
    c. Resolve poll headers
    d. Parse poll method
    e. Send poll request via `self.send_request()` with retry
    f. Check poll response status — non-2xx fails immediately (D-15)
    g. Parse poll response body as JSON — non-JSON fails immediately (D-16)
    h. Check `completed_when.evaluate(&poll_json)` — if true (D-10 check first):
       - Scan `self.response.success.body` for `<poll_response.*>` placeholders
       - Extract each from poll JSON into variables
       - Resolve success body template
       - Parse success header
       - Return success ExecutionResult
    i. Check `failed_when` (if Some) `.evaluate(&poll_json)` — if true:
       - Scan `self.response.failed` body (if present) for `<poll_response.*>` placeholders (D-25)
       - Extract each from poll JSON into variables
       - Resolve failed body template
       - Parse failed header
       - Return failure ExecutionResult with error "failed_when condition matched: {path} {operator} {value}"
    j. Log poll iteration, continue loop
