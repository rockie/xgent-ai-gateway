Extend the existing sample_service.rs to add three new endpoints per D-05. Keep the existing POST /execute endpoint unchanged.

**POST /sync endpoint:**
- Accept POST with any body content
- Return JSON: `{"status": "ok", "result": {"text": "<echoed payload>", "length": <payload length>}}`
- Content-Type: application/json
- Print log line: `[task_id={X-Task-Id}] POST /sync received`

**POST /async/submit endpoint:**
- Accept POST with any body content
- Generate a UUID job_id (use `uuid::Uuid::now_v7().to_string()`)
- Store the job in an `Arc<Mutex<HashMap<String, AsyncJob>>>` shared state, where `AsyncJob` is a struct with fields: `payload: String, created: Instant, polls: u32`
- Return JSON: `{"job_id": "<uuid>", "status": "accepted"}`
- Print log line: `[job_id={job_id}] POST /async/submit received`

**GET /async/status/:id endpoint:**
- Look up job_id from the URL path (parse from `/async/status/{id}`)
- If not found, return 404 with `{"error": "job not found"}`
- Increment the job's `polls` counter
- If `polls >= 3`, return: `{"job_id": "<id>", "status": "completed", "result": {"text": "<original payload>", "processed": true}}`
- If `polls < 3`, return: `{"job_id": "<id>", "status": "processing", "progress": polls * 33}`
- Print log line: `[job_id={id}] GET /async/status (poll #{polls})`

**Implementation notes:**
- The shared state `Arc<Mutex<HashMap<String, AsyncJob>>>` is created in main() and cloned into the handle function closure
- Use `uuid` crate which is already a dependency of the gateway crate
- For path parsing: split `req.uri().path()` on `/` and match segments
- The existing handle() function signature changes to take shared state as a parameter. Wrap it: `let state = Arc::clone(&jobs); service_fn(move |req| { let st = Arc::clone(&state); async move { handle(req, st).await } })`
- Add `use std::sync::Mutex as StdMutex;` (not tokio::sync::Mutex since we only hold it briefly)
- Add `use std::time::Instant;`
- Update the startup print to show all endpoints:
  ```
  sample-service listening on {bind}:{port}
    POST /execute   -- echo payload (with optional delay)
    POST /sync      -- echo payload in JSON wrapper
    POST /async/submit     -- create async job
    GET  /async/status/:id -- poll job status (completes after 3 polls)
  ```

#!/bin/bash
# echo.sh - Simple echo script for xgent CLI execution examples.
# Reads input from args or stdin and outputs a JSON result.

  INPUT="$1"
else

fi
