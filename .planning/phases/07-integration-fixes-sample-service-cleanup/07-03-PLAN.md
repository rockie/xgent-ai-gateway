**Create gateway/examples/sample_service.rs (per D-01, D-02, D-03, D-04):**

A standalone single-file example using hyper 1.x directly. No imports from the gateway crate. Plain HTTP only.

```rust
//! Sample service for xgent-gateway end-to-end testing.
//!
//! Echoes the task payload back as the result. Optionally simulates processing
//! delay when the `X-Simulate-Delay-Ms` header is present.
//!
//! Usage:
//!   cargo run -p xgent-gateway --example sample_service
//!   cargo run -p xgent-gateway --example sample_service -- --port 9090
//!
//! Pair with the runner agent:
//!   cargo run -p xgent-gateway --bin xgent-agent -- \
//!     --service-name my-service --token <token> --dispatch-url http://localhost:8090/execute
```

The implementation must:

1. Parse CLI args with `clap`:
   - `--port` (default: `8090`, matching agent's default `--dispatch-url`)
   - `--bind` (default: `0.0.0.0`)

2. Create a hyper 1.x HTTP server using `hyper_util::server::conn::auto::Builder`:
   - Bind a `tokio::net::TcpListener` to `{bind}:{port}`
   - Accept connections in a loop
   - For each connection, spawn a task that serves the request

3. Request handler:
   - Match on `POST /execute` -- the path the runner agent dispatches to
   - Extract `X-Task-Id` header for logging (optional, just for tracing output)
   - Extract `X-Simulate-Delay-Ms` header, parse as `u64`, sleep that duration if present
   - Collect the request body bytes using `http_body_util::BodyExt::collect`
   - Return HTTP 200 with the same bytes as the response body (echo)
   - For any other method/path, return HTTP 404

4. Use these exact crate imports (all available in gateway Cargo.toml):
   - `hyper` (1.x) -- `Request`, `Response`, `StatusCode`
   - `hyper_util` -- `server::conn::auto::Builder`, `rt::TokioExecutor`, `rt::TokioIo`
   - `http_body_util` -- `Full`, `BodyExt`
   - `tokio` -- runtime, `net::TcpListener`, `time::sleep`
   - `clap` -- `Parser` derive
   - `bytes` -- `Bytes`

5. Print startup message: `println!("sample-service listening on {bind}:{port}")` and `println!("  POST /execute -- echo payload (with optional delay)")`.

6. The example must NOT import anything from `xgent_gateway`. It is fully standalone per D-02.

**IMPORTANT implementation detail for hyper 1.x service:**

The hyper 1.x service trait requires implementing `hyper::service::Service` or using a closure. The simplest approach for an example:

```rust
use hyper::service::service_fn;
use std::convert::Infallible;

async fn handle(req: Request<hyper::body::Incoming>) -> Result<Response<Full<Bytes>>, Infallible> {
    // ... handler logic
    Ok(response)
}

// In the accept loop:
let service = service_fn(handle);
builder.serve_connection(io, service).await;
```

Use `hyper::body::Incoming` as the request body type and `http_body_util::Full<Bytes>` as the response body type.
