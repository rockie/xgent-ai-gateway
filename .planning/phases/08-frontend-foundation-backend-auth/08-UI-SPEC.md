## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label |
| sm | 8px | Compact element spacing, form field gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, sidebar item vertical rhythm |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Login page section breaks, page-level vertical margins |
| 3xl | 64px | Login split-layout gutter |

Exceptions: Sidebar width expanded = 256px (16rem), sidebar width collapsed = 48px (icon-only mode with 4px horizontal padding). Touch targets for sidebar nav items = 40px height minimum.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (0.875rem) | 400 (regular) | 1.5 |
| Label | 12px (0.75rem) | 400 (regular) | 1.5 |
| Heading | 20px (1.25rem) | 600 (semibold) | 1.2 |
| Display | 28px (1.75rem) | 600 (semibold) | 1.2 |

**Usage rules:**
- Body (14px): All paragraph text, table cells, form inputs, sidebar nav labels, toast messages
- Label (12px): Form labels, caption text, auto-refresh interval display, metadata timestamps
- Heading (20px): Page titles ("Dashboard", "Services", "Tasks", "Credentials"), card headings
- Display (28px): Login page project name ("xgent gateway"), empty state headings on placeholder pages

**Weight constraint:** Only 400 and 600 used. No bold (700) anywhere.

## Color

Uses shadcn/ui CSS variable system with Tailwind v4. Colors defined via `@theme` in `index.css`. Dark mode is default (D-23).

| Role | Dark Mode | Light Mode | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `--background: 240 10% 3.9%` (zinc-950) | `--background: 0 0% 100%` (white) | Page background, content area |
| Secondary (30%) | `--card: 240 10% 5.9%` (zinc-900) | `--card: 0 0% 98%` (zinc-50) | Sidebar background, cards, login left panel, header bar |
| Accent (10%) | `--primary: 0 0% 98%` (zinc-50) | `--primary: 240 5.9% 10%` (zinc-900) | See reserved-for list below |
| Destructive | `--destructive: 0 62.8% 30.6%` (red-900) | `--destructive: 0 84.2% 60.2%` (red-500) | Logout button hover, future delete confirmations |

Accent reserved for:
- "Sign in to gateway" button on login page (primary CTA)
- Active sidebar navigation item indicator
- Auto-refresh active state indicator in header
- Toast success icon color
- "Retry request" button in error states

**Not accent:** Sidebar icons (use `--muted-foreground`), form borders (use `--border`), body text (use `--foreground`).

## Layout Specifications

### Login Page (split layout per D-16)

**Primary focal point:** The login form card on the right panel is the primary focal point. The user's eye should land on the "Sign in to gateway" heading and the username input first. The brand panel on the left is secondary context.

```
+---------------------------+---------------------------+
|                           |                           |
|     [brand panel]         |     [login form]          |
|                           |                           |
|     xgent gateway         |     Sign in to gateway    |
|     (display 28px)        |     [username input]      |
|                           |     [password input]      |
|     project tagline       |     [sign in to gateway]  |
|     (body 14px, muted)    |                           |
|                           |     [error alert]         |
|                           |                           |
+---------------------------+---------------------------+
```

- Two equal columns (50/50) at 1280px+
- Brand panel uses secondary background color
- Login form panel uses dominant background color
- On viewports below 1280px, brand panel is hidden (login form goes full width)
- Login form card is vertically and horizontally centered in its panel
- Maximum form width: 384px (24rem)

### Authenticated Shell (sidebar + header + content)

```
+--------+--------------------------------------------------+
| SIDEBAR|  HEADER [auto-refresh] [dark mode] [user menu]   |
|        +--------------------------------------------------+
| [nav]  |                                                  |
|        |  CONTENT AREA                                    |
| Dash   |  (page-specific content, skeletons, errors,     |
| Svc    |   empty states render here)                      |
| Tasks  |                                                  |
| Creds  |                                                  |
|        |                                                  |
+--------+--------------------------------------------------+
```

- Sidebar expanded: 256px wide (16rem)
- Sidebar collapsed: 48px wide (icon-only, tooltips on hover)

## Interaction States

### Login Form
- **Idle:** Inputs unfocused, CTA button enabled
- **Focused:** Input border changes to `--ring` color
- **Submitting:** CTA button shows loading spinner, inputs disabled
- **Error:** Alert banner appears below the form with error copy. Inputs remain filled. CTA re-enables.
- **Success:** Redirect to originally requested page (or `/` dashboard). Toast: "Signed in successfully."

### Sidebar Navigation
- **Default:** Icon + label, muted foreground color
- **Hover:** Background changes to `--accent` (subtle highlight)
- **Active:** Background `--accent`, foreground `--accent-foreground`, left border indicator (2px primary color)
- **Collapsed:** Icons only, tooltip on hover shows label text

### Auto-Refresh Control
- **Off (default for Phase 8):** Dropdown shows "Off", no visual indicator
- **Active:** Dropdown shows interval (e.g., "15s"), rotating refresh icon animates in header
- **Paused:** Dropdown shows interval but icon is static, label shows "Paused"

### Dark Mode Toggle
- **Dark (default):** Moon icon displayed
- **Light:** Sun icon displayed
- **Toggle:** Instant switch, no transition animation. Preference saved to localStorage immediately.

### Error State (inline)
- **Layout:** Alert component within content area. Sidebar and header remain visible and functional.
- **Content:** AlertCircle icon + heading "Something went wrong" + error message + "Retry request" button
- **Retry:** Button triggers query refetch. Shows loading state during retry.

### Empty State (placeholder pages)
- **Layout:** Centered in content area, vertically offset to upper third
- **Content:** Icon (Construction/Wrench from lucide-react) + "Coming Soon" heading + description text
- **No action button** for placeholder pages
