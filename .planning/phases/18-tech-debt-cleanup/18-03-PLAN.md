Refactor `init_tracing` (lines 39-107) from a 4-arm match into composable layer construction.

Current pattern: 4 match arms for `(file/no-file, json/text)` each building the full subscriber independently, with the file_layer always being JSON regardless of the `is_json` flag (the stdout layer varies).

Replace with:
```rust
fn init_tracing(config: &LoggingConfig) -> Option<tracing_appender::non_blocking::WorkerGuard> {
    let env_filter = tracing_subscriber::EnvFilter::try_from_default_env()
        .unwrap_or_else(|_| tracing_subscriber::EnvFilter::new("info"));

    let is_json = config.format == "json";

    // File layer is always JSON format (structured logs for log aggregation)
    let (file_layer, guard) = if let Some(ref file_path) = config.file {
        let file = std::fs::OpenOptions::new()
            .create(true)
            .append(true)
            .open(file_path)
            .expect("Failed to open log file");
        let (non_blocking, guard) = tracing_appender::non_blocking(file);
        let layer = tracing_subscriber::fmt::layer()
            .json()
            .with_target(true)
            .with_writer(non_blocking);
        (Some(layer), Some(guard))
    } else {
        (None, None)
    };

    if is_json {
        let stdout_layer = tracing_subscriber::fmt::layer()
            .json()
            .with_target(true);
        tracing_subscriber::registry()
            .with(env_filter)
            .with(stdout_layer)
            .with(file_layer)
            .init();
    } else {
        let stdout_layer = tracing_subscriber::fmt::layer()
            .with_target(true);
        tracing_subscriber::registry()
            .with(env_filter)
            .with(stdout_layer)
            .with(file_layer)
            .init();
    }

    guard
}
```

Note: We still need two branches for stdout (json vs text) because `.json()` changes the layer type. But the file layer setup is deduplicated. This reduces from 4 arms to 2 branches + shared file setup. The `Option<Layer>` works because tracing_subscriber supports `Option<L>` where `None` is a no-op layer.

IMPORTANT: The tracing-subscriber `Option<Layer>` trick works with `.with(file_layer)` because `Option<L: Layer<S>>` implements `Layer<S>`. However, the JSON file layer type differs from what the compiler infers for `None`. To handle this, use `tracing_subscriber::fmt::layer().json().with_target(true).with_writer(non_blocking)` boxed: wrap in `.boxed()` or use `Option<Box<dyn Layer<_> + Send + Sync>>`. Actually, the simpler approach: since both branches of the if/else produce the same Option type (they're the same layer constructor), this works directly.

If the types don't align, fall back to the two-branch approach (json stdout vs text stdout) but factor out the file layer construction into a helper. The key improvement is not repeating the file setup 4 times.
