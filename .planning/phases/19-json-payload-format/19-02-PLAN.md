3. Remove the `base64::Engine::decode(...)` block that decoded base64 payload
4. Add JSON serialization: `let payload_json = serde_json::to_string(&req.payload).map_err(|e| GatewayError::InvalidRequest(format!("failed to serialize payload: {e}")))?;`
5. Pass `payload_json` (String) to `queue.submit_task(&service, payload_json, req.metadata)`
6. Remove `use base64::Engine as _;` import if present
7. Add `use serde_json;` if not already imported (likely available through serde re-export)

**result.rs:**
1. Change `GetTaskResponse.result` type from `pub result: Option<String>` to `pub result: Option<serde_json::Value>`
2. Replace the base64 encoding block:
   ```rust
   // Before: Some(base64::Engine::encode(&base64::engine::general_purpose::STANDARD, &status.result))
   // After:
   let result = if status.result.is_empty() {
       None
   } else {
       Some(serde_json::from_str(&status.result).unwrap_or(serde_json::Value::String(status.result.clone())))
   };
   ```
   The `unwrap_or` fallback handles any non-JSON data gracefully (wraps as string value).
3. Remove `use base64::Engine as _;` import if present

**admin.rs:**
1. Change `TaskDetailResponse` fields:
   - `pub payload: String` (with comment `// base64-encoded`) -> `pub payload: serde_json::Value`
   - `pub result: String` (with comment `// base64-encoded`) -> `pub result: serde_json::Value`
2. In the task detail handler, replace the base64 encoding:
   ```rust
   // Before:
   // let payload_b64 = base64::engine::general_purpose::STANDARD.encode(&status.payload);
   // let result_b64 = if status.result.is_empty() { String::new() } else { base64::...encode(&status.result) };

   // After:
   let payload_json: serde_json::Value = if status.payload.is_empty() {
       serde_json::Value::Null
   } else {
       serde_json::from_str(&status.payload).unwrap_or(serde_json::Value::String(status.payload.clone()))
   };
   let result_json: serde_json::Value = if status.result.is_empty() {
       serde_json::Value::Null
   } else {
       serde_json::from_str(&status.result).unwrap_or(serde_json::Value::String(status.result.clone()))
   };
   ```
3. Use `payload_json` and `result_json` in the `TaskDetailResponse` construction
4. Remove `use base64::Engine as _;` import from the top of the file
5. Check if `base64` is used anywhere else in admin.rs (it should not be after this change). If not, remove the import entirely.

**grpc/submit.rs:**
The gRPC `submit` handler currently passes `req.payload` (was `Vec<u8>` from proto bytes) to `queue.submit_task()`. With proto changed to `string`, `req.payload` is now `String`. The `queue.submit_task()` now takes `String`. This should "just work" -- but verify no intermediate conversions exist. If there's any `.to_vec()` or byte conversion, remove it.

The `get_task_status` response now has `result` as `String` (proto string). The handler that builds `GetTaskStatusResponse` from `TaskStatus` should pass `status.result` directly (String -> String).

**grpc/poll.rs:**
1. In `poll_tasks` stream handler: `task_data.payload` is now `String`. The `TaskAssignment` proto message `payload` is now `string`. Assignment construction should pass the String directly without conversion.
2. In `report_result` handler: `req.result` is now `String` (proto string). Pass directly to `queue.report_result(...)` which now takes `String`.

**cli_executor.rs:**
Change every `ExecutionResult` construction to use `String` for result field:
- All `result: Vec::new()` -> `result: String::new()`
- `result: result_bytes` (where result_bytes was from `resolve_response_body`) -> `result: result_string` (resolve_response_body now returns String)
- `result: bytes` (similar pattern) -> rename to `result: result_str` or similar, adjust the variable that feeds it
- In test helpers (`make_assignment` or similar): change `payload: b"...".to_vec()` to `payload: "...".to_string()`

**sync_api_executor.rs:**
Same pattern as cli_executor:
- All `result: Vec::new()` -> `result: String::new()`
- `result: result_bytes` -> `result: result_string` (from resolve_response_body which now returns String)
- `result: bytes` -> adjust accordingly
- In tests: update `make_assignment` payload from bytes to String

**async_api_executor.rs:**
Same pattern:
- All `result: Vec::new()` -> `result: String::new()`
- Result from resolve_response_body is now String, assign directly
- In tests: update assignment payloads to String

**bin/agent.rs:**
The agent binary builds `ReportResultRequest` with `result: exec_result.result`. Since `exec_result.result` is now `String` and the proto `result` field is now `string` (mapped to Rust String by prost), this should work directly. Verify no `.to_vec()` or byte conversion exists.

**General note:** After proto regeneration, prost maps `string` fields to Rust `String` and `bytes` to `Vec<u8>`. All call sites that previously used `Vec<u8>` for these fields now use `String`. The compiler will flag any remaining mismatches.
