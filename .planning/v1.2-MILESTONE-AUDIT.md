## Tech Debt

### Phase 15: Dead Code
- `ExecutionResult.headers` field is populated by all three executors but never forwarded in `ReportResultRequest` — the proto message has no `headers` field. Headers are collected but silently discarded.

### All Phases: Pre-existing Warning
- `unused_assignments` compiler warning on `has_in_flight` in `agent.rs` (graceful drain logic) — pre-dates v1.2 work.
- Pre-existing clippy warnings in `http/admin.rs`, `metrics.rs`, `types.rs` — not in v1.2 files.

### Documentation Discrepancies
- REQUIREMENTS.md references `agent.toml` in CFG-01, EXMP-01, EXMP-02, EXMP-03 — implementation uses `agent.yaml`
- REQUIREMENTS.md CFG-04 states `<meta.key>` — implementation uses `<metadata.key>` per design decision D-05

### Advisory (non-blocking)
- Boolean values in async-api poll response are coerced to strings by `extract_json_value` (e.g., `true` → `"true"`). By design.
- CLI example configs use relative path to `echo.sh` — requires agent to run from repo root. Documented in README.

**Total:** 5 tech debt items + 2 doc discrepancies + 2 advisory (0 blockers)
